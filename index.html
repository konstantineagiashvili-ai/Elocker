⚙️ Backend-ის დეტალური დეველოპერული სამუშაოები

1. მონაცემთა მოდელირება და ინტეგრაცია

  • Product Schema განახლება: ბაზაში პროდუქტის მოდელს უნდა დაემატოს ველები: height, width, depth, weight.
  • ERP/1C სინქრონიზაცია: არსებული იმპორტის სკრიპტების (Cron/Listener) მოდიფიკაცია, რათა ERP-დან წამოიღოს და განაახლოს ზომის პარამეტრები.
  • Locker Entity / Warehouse: ლოქერების სისტემაში აღრიცხვა ვირტუალურ საწყობებად (warehouseId), რომ სწორად იმუშაოს მარაგების კონტროლმა.

2. მოცულობის ალგორითმი და ვალიდაცია

  • მოცულობის კალკულაციის სერვისი: ლოგიკა, რომელიც ითვლის კალათის ჯამურ მოცულობას Σ(qty * (H*W*D)) და ითვალისწინებს ტარების ფაქტორს.
  • Locker Box Mapping: კალკულატორის შედეგის დაკავშირება ლოქერის უჯრების ტიპებთან (მაგ: < 20L = 1 უჯრა, < 40L = 2 უჯრა).
  • Checkout Middleware: POST /orders ენდფოინთზე მკაცრი ვალიდაციის დამატება. თუ მოცულობა სცდება მაქსიმალურ ლიმიტს, დააბრუნოს HTTP 4xx (მაგ: "Locker Capacity Exceeded").

3. eLocker API ინტეგრაცია

  • HTTP Client: eLocker-ის API-სთან საკომუნიკაციო მოდულის დაწერა (Bearer token/Auth მართვა).
  • Create Order Flow: შეკვეთის წარმატებით დადასტურებისას, ვენდორისთვის POST მოთხოვნის გაგზავნა (გადაეცემა მომხმარებლის ტელეფონი, შიდა Order ID და დათვლილი Box Size).
  • Cancel Order Flow: თუ მომხმარებელი ან ადმინი აუქმებს შეკვეთას, გაიგზავნოს მოთხოვნა ლოქერიდან ამოშლის შესახებ.

4. Webhook & ნოტიფიკაციები

  • Webhook Listener ენდფოინთი: (მაგ. POST /api/webhook/elocker) ვენდორიდან სტატუსების მისაღებად (Deposited, Collected).
  • სტატუსების მართვა: მიღებული Webhook-ების საფუძველზე სისტემაში შეკვეთის სტატუსების შეცვლა და ლოგირება.
  • Event Trigger: "Deposited" სტატუსის მიღებისას შიდა Notification Service-ისთვის ივენთის გაშვება მომხმარებლისთვის OTP კოდის გასაგზავნად.

💻 Frontend-ის დეტალური დეველოპერული სამუშაოები

1. UI / UX კალათის მართვა (Cart)

  • Volume State Calculator: Redux/Zustand-ში სელექტორის დამატება, რომელიც ლოკალურად ითვლის კალათის მოცულობას პროდუქტის ზომებზე დაყრდნობით.
  • Progress Bar კომპონენტი: კალათაში ახალი ვიზუალური ინდიკატორის შექმნა, რომელიც რეალურ დროში აჩვენებს დაკავებულ ადგილს (მაგ. w-[80%] bg-orange-400).
  • Warning/Error ინდიკაცია: როცა ლიმიტი უახლოვდება მაქსიმუმს ზოლის ფერის შეცვლა, ხოლო შევსებისას წითლად ჩვენება და საინფორმაციო ტექსტის გამოტანა.

2. შეზღუდვები და ვალიდაცია ფრონტზე

  • ღილაკების ბლოკირება: <ProductCard> და <CartItem> დონეზე "+" ღილაკის და AddToCart მოქმედების disabled სტატუსში გადაყვანა, თუ პროდუქტის დამატება აჭარბებს ჯამურ მოცულობას.
  • Toast / Notification: მომხმარებლის ინფორმირება ამომხტარი შეტყობინებით (მაგ: "პროდუქტი ვერ დაემატება, ლოქერის მოცულობა ამოწურულია").

3. Checkout პროცესის მოდიფიკაცია

  • Locker Delivery Option: Delivery ტიპებში "Locker"-ის დამატება (Standard Delivery-ს და Pickup-ის გვერდით).
  • Locker Selector: Google Maps ინტეგრაცია ან Dropdown, საიდანაც მომხმარებელი ირჩევს კონკრეტულ ლოქერს (გადაეცემა lockerId ბექენდს).
  • Payload Update: Checkout მოთხოვნაში ახალი ველების გატანის უზრუნველყოფა.

4. User Profile & Order History

  • Active Orders: შეკვეთების ისტორიაში Locker ტიპის შეკვეთებისთვის სპეციალური ვიზუალის შექმნა (მაგ. "ელოდება ლოქერში").
  • OTP / QR კოდის ჩვენება: თუ სტატუსი არის "Locker_Deposited", მომხმარებლის ეკრანზე გამოჩნდეს დიდი შრიფტით გასატანი PIN კოდი და ლოქერის ინსტრუქცია.
